# Defining the Schema üç± (contd.)

The paragraph node also includes methods that let Prosemirror parse/render it as a `p` tag in the actual DOM.

```
paragraph: {
  group: "block",
  content: "inline*",
  parseDOM: [{ tag: "p" }],
  toDOM() {
    return ["p", 0];
  },
}
```

Take a look at the [serialize/deserialize](https://prosemirror.net/docs/guide/#schema.serialization_and_parsing) section in the docs to know more about these functions.

Let's play around with this a little. Try changing the `p` tags in parseDom and toDom to heading tags (`h1` or `h2`) and see what happens!
